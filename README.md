# Development Process 

Developers work in their own trees, then submit pull requests when they think their feature or bug fix is ready.

The patch will be accepted if there is broad consensus that it is a good thing. Developers should expect to rework and resubmit patches if they don't match the project's coding conventions (see coding.txt) or are controversial.

The master branch is regularly built and tested, but is not guaranteed to be completely stable. Tags are regularly created to indicate new stable release versions of LiteDoge.

Feature branches are created when there are major new features being worked on by several people.

From time to time a pull request will become outdated. If this occurs, and the pull is no longer automatically mergeable; a comment on the pull will be used to issue a warning of closure. 

Issues with no commits will be given a similar warning, and closed after 15 days from their last activity. Issues closed in this manner will be labeled 'stale'.

                                <h2 id="wallet">LiteDoge Wallet Setup Guide</h2>

                                <p class="media-cell-desc">Difficulty-level: Easy <i class="pw-icon-star"></i></p>

                                <p>LiteDoge is a digital asset that uses cryptography to secure the transactions and asset transfers. It uses a network of peer-to-peer nodes to verify and record transactions in a public distributed ledger.</p>

                                <p>LiteDoge wallet is a program that participates the p2p network as a full node by validating transactions and blocks from other full nodes. It’s also a wallet to store your LDOGEs, so you must take the same precautions you would when using any cryptocurrency wallet. This guide walks you trough the most basic stuff, but please take your time to educate yourself before using LiteDoge to any real financial transactions.</p>

                                <p>
                                    <a class="button blue tutorial" href="https://web.archive.org/web/20180308162741/https://github.com/vashshawn/LDOGE/releases/download/v3.4.1.1/litedoge-qt.3.4.1.1.Win.zip">Download LiteDoge Wallet for Windows »</a>

                                    <a class="button red tutorial" href="https://web.archive.org/web/20180308162741/https://github.com/vashshawn/LDOGE/releases/download/v3.4.1.1/LiteDoge-Qt-Mac-v3.4.1.1.dmg">Download LiteDoge Wallet for Mac OS X »</a>

                                    <a class="button green tutorial" href="https://web.archive.org/web/20180308162741/https://github.com/vashshawn/LDOGE/wiki/Ubuntu-Desktop-Wallet-Installation-Guide" target="_blank">Ubuntu Desktop Installation Guide »</a>

                                </p>

                                <p>After downloading and installing the wallet, program creates the data directories and <span class="code">wallet.dat</span>, and begins downloading the LiteDoge blockchain. Location of the data directory depends on your operating system:</p>

                                <pre class="code"># Windows XP
C:\Documents and Settings\&lt;put-your-username-here&gt;\Application Data\LiteDoge\

# Windows Vista, Windows 7, Windows 8/8.1 and Windows 10
C:\Users\&lt;put-your-username-here&gt;\AppData\Roaming\LiteDoge\

# Linux
/home/&lt;put-your-username-here&gt;/.litedoge/

# Max OS X
/Users/&lt;put-your-username-here&gt;/Library/Application Support/LiteDoge/</pre>

                                <h4>Optional: Bootstrapping the blockchain</h4>

                                <p>
                                    <a class="button green tutorial" href="https://web.archive.org/web/20180308162741/https://mega.nz/#!DxICFL7C!ZHUTb0SDz3LPQq4PobHbrOF46RE8Q5ZgJbjb-MqIHJI">Download bootstrap.dat »</a>
                                </p>

                                <p>If you are in hurry, you can fast forward syncing by downloading the blockchain and bootstrapping it. Download the <span class="code">bootstrap.dat</span> file and unpack it replacing everything except <span class="code">wallet.dat</span> and <span class="code">litedoge.conf</span> in your data directory, and restart the wallet. This time syncing should take only less than hour to complete!</p>

                                <p>When your wallet is fully synchronized, you can receive LDOGE and start staking.</p>

                                <h4>Staking LiteDoge</h4>

                                <p>New coins (LDOGE) are created as a reward for running a node in a process known as staking. You can earn LiteDoge by validating block transactions. Keep your coins for at least 8 hours in your wallet and they will be eligible for staking. The time needed to get stake reward depends on the number of coins you hold in your wallet.</p>
